This is the on-ramp most sessions start from: before you can change a pattern you have to locate it. These three pieces are Phase-1 work (`core-model.md`) and they're the safest, most universally useful part of the skill — pure psychoeducation and self-reflection, no contraindications. Deliver each per `walkthrough-protocol.md`: one step per turn, end on a question, wait. The schema and mode names and their log tags are in `core-model.md`.

## 1 · Schema identification (which patterns are mine)

**Reach for it when:** the user knows something keeps repeating but can't name it; a new session where the schema profile isn't yet on record; or any time the pattern in the room is fuzzy. This is **not** a scored questionnaire (the YSQ is a clinical instrument) — it's guided self-reflection toward naming an Early Maladaptive Schema.

**Walk it through:**
1. Start from a concrete repeating moment, not a generality. Ask: "Give me one recent time you felt that old, familiar bad feeling — walk me to the exact moment." (A felt sense of *familiarity* — "here it is again" — is the tell that a schema, not just a one-off mood, is firing.)
2. Pull the felt belief about self/others underneath it. Ask: "In that moment, what did it say about you, or about what people are like?" (e.g. "I'm too much", "they'll leave", "I can't trust this".)
3. Trace it back. Ask: "How far back does that feeling go — when's the earliest you remember feeling exactly that?" Link the adult moment to the early origin (gently; don't excavate trauma, just locate the root).
4. Name the schema together, in their words first, then map it to a named EMS and its domain (`core-model.md`). Ask: "Does it fit best as [name the one or two likely schemas in plain language]?" Let them confirm or correct.
5. Name the unmet need behind it (one of the five). Ask: "So the thing you needed back then and didn't reliably get was ___ — does that land?" This reframes the schema as an understandable adaptation, not a defect.
6. Rate how strongly the belief grips (0-100%) for the log and for later re-rating.

**If it stalls:** can't find a memory → work forward from the body sensation instead ("where do you feel it, and when does your body do that?"). Too many patterns at once → pick the one tied to the most acute recent moment and park the rest. Stays purely intellectual / no felt sense → that flatness is often a Detached coping mode; switch to mode identification (2) and name the detachment first.

## 2 · Mode identification (which state am I in)

**Reach for it when:** the user describes flipping ("I just go cold", "I turn into a different person", "something takes over"), or you need to know which mode is live before choosing a change technique. The mode is usually what you actually work with.

**Walk it through:**
1. Catch the shift. Ask: "When it hit, what changed in you — what did you feel, want to do, and actually do?" Listen for which mode (`core-model.md`): a hurt/scared Child, an attacking Critic voice, a numbing/withdrawing or pleasing coping move, an explosive Angry Child.
2. Name it plainly and let them re-name it personally. Ask: "That sounds like a part that [protects / attacks / shuts down] — what would you call it?" (Personal labels — "the wall", "the drill sergeant", "little me" — stick better than clinical terms.)
3. Find its job. For coping modes especially: "What is that part trying to protect you from? What happens if it drops its guard?" (The Detached Protector guards the Vulnerable Child; honor that before trying to get past it.)
4. Spot the parent/critic voice if present. Ask: "Whose voice does the harsh part sound like? What does it say, word for word?" (Distinguishes Punitive — "you deserve to suffer" — from Demanding — "you must do better".)
5. Locate the Vulnerable Child underneath the coping/critic modes. Ask: "Under the [wall / critic], how old does the hurt feel, and what does that younger part feel?"
6. Note the modes for the log and for the mode map (3).

**If it stalls:** "I feel nothing" → that *is* the mode (Detached Protector/Self-Soother); validate its protective job, don't push through it. Can't tell Punitive from Demanding → go by the message: punishment/worthlessness vs. higher-performance demand. Flooded by the Child → ground first (`safety.md`) before continuing.

## 3 · Case conceptualization — building the mode map

**Reach for it when:** a couple of schemas/modes are identified and you want the picture of how they chain together — the running map that makes every later session start warm. This is the self-help version of the schema-therapy mode map; it lives in `CLAUDE.md` (`logging.md`) and gets refined over time.

**Walk it through:**
1. Pick a recent representative episode (a triggered loop, start to finish).
2. Lay out the **sequence** one link at a time, asking at each: trigger → which mode activated first → what it triggered next → where it ended. Ask: "So it started with ___; then what came up; then what did that lead to?" (e.g. *partner went quiet → Vulnerable Child, abandonment panic → Detached Protector, went cold → Angry Child, picked a fight*.)
3. Identify the **dyads**: which critic/parent mode lights up which Child. Ask: "When the little one is hurt, which harsh voice shows up to it?"
4. Mark where the **Healthy Adult** already shows up, however small. Ask: "Was there any moment, even a second, where a wiser/calmer part could see all this? When does that part tend to be strongest?"
5. Sketch it back to them as a simple map — modes as parts, arrows as the sequence, the dominant ones bigger — and check it fits. Ask: "Does this picture match how it actually goes?"
6. Save it to `CLAUDE.md` "Mode map"; treat it as a living draft and revise it as sessions sharpen it.

**If it stalls:** the loop is too tangled → just capture the first two links (trigger → first mode) and build the rest over future sessions. They resist the "parts" framing → drop the language, keep the function ("the bit of you that shuts down", "the bit that attacks you"). Once the map is clear, route from it: usually name/validate the coping mode (`healthy-adult.md`), then access the Child experientially (`experiential.md`), then consolidate cognitively and behaviorally (`cognitive.md`, `behavioral.md`).
