## How to route

The user won't name a protocol, they'll name a state or describe a stuck day ("I haven't left the flat in a week", "nothing's fun anymore", "I just lie there and scroll"). Your job: from what they say plus a concrete picture of what they're actually doing, identify the **problem area**, then reach for that area's **first-line technique** and open its file. Pick one technique to start, not a menu. Re-route freely if their answers point elsewhere.

Always do the safety scan (`safety.md`) and get a mood + 0-100 intensity + a concrete picture of the avoidance/withdrawal pattern first (`SKILL.md` flow steps 1-2).

Almost everything in BA starts the same way: **monitor what's actually happening, then schedule small valued/mastery activities, then target the avoidance keeping the cycle going.** The condition files below add the specialized layer on top of that universal spine.

## Front door: what the user says → problem area

| What they bring | Problem area | File |
|---|---|---|
| Flat, heavy, withdrawn, doing less, "what's the point", stopped going out, isolating, lost their routine | Depression / low mood | `depression.md` |
| "Nothing's enjoyable / I feel nothing", lost interest in things they used to love, can't even want anything, blunted | Anhedonia | `anhedonia.md` |
| Withdrawn and avoidant **after a loss** (death, breakup, role loss), avoiding reminders, "if only" loops | Grief / bereavement | `grief.md` |
| Avoiding situations out of fear/dread **alongside** low mood; anxiety and depression tangled together | Anxiety comorbid with depression | `anxiety-overlap.md` |
| Pain pulling activity and mood down, boom-and-bust cycle, stopped doing things because of the body; or low mood alongside a medical illness / new motherhood / substance recovery | Chronic pain & medical comorbidity | `pain-medical.md` |
| Stuck mainly because of avoidance of one situation or task (not fear-driven) — the avoidance itself is the engine | Behavioral avoidance | `core-techniques.md` (TRAP/TRAC, anti-avoidance) |
| Stuck in repetitive "why am I like this" / "if only" thinking that crowds out doing anything | Rumination as avoidance | `core-techniques.md` (rumination-as-avoidance) |
| Daily structure collapsed — erratic sleep, skipping meals, no shape to the day | Routine disruption | `core-techniques.md` (routine establishment), then `depression.md` |
| An older adult, adolescent, or a group / telephone / app context; or needs culturally adapted delivery | Special population / delivery format | `special-populations.md` |
| Panic surges, a specific phobia, OCD compulsions, or standalone social/health/generalized anxiety with no depressive withdrawal | Out of scope for BA | say so plainly; these want cognitive/exposure work (a CBT skill), not BA. See `safety.md` for crisis flags. |
| Generically "stuck" / "can't get going", no clear flavour yet | Needs triage | ask one locating question (what does a day actually look like?), then route — usually depression, anhedonia, or pure avoidance |

If two rows fit, ask one clarifying question to pick, or start with the one tied to the most acute state. Anhedonia vs ordinary low mood: if they can still want things but aren't doing them, route `depression.md`; if wanting itself is gone ("I can't even look forward to anything"), route `anhedonia.md`.

## Problem area → first-line technique → file

The subset to reach for **first** in each area. Each technique lives in the listed file; the universal toolkit is in `core-techniques.md`. (Section numbers below match the technique numbering in `core-techniques.md` and `anhedonia.md`.)

| Problem area | First-line technique(s) | Files |
|---|---|---|
| Depression / low mood | Activity Monitoring → Activity Scheduling → TRAP/TRAC → Routine Establishment; Graded Task Assignment if scheduling stalls | `depression.md` + `core-techniques.md` |
| Severe / treatment-resistant depression | Monitoring alone first, then **micro-step** Graded Task Assignment; Contingency Management | `depression.md` + `core-techniques.md` (read `safety.md`) |
| Anhedonia | BATA **Dabbling** → **Savoring**; Values Assessment; mastery-first scheduling | `anhedonia.md` + `core-techniques.md` |
| Grief / bereavement | Values Clarification beyond the lost role, Activity Scheduling to re-engage surviving roles, Anti-Avoidance for grief-stimulus avoidance, Rumination-as-Avoidance | `grief.md` + `core-techniques.md` |
| Anxiety comorbid with depression | TRAP/TRAC for anxiety-avoidance, graded approach activities, Behavioral Experiments | `anxiety-overlap.md` + `core-techniques.md` |
| Chronic pain + low mood | Values-based + time-quota graded activity (pacing, not pain-contingent), activity scheduling | `pain-medical.md` + `core-techniques.md` |
| Other medical / SUD / postpartum | Values Clarification, scheduling around the constraint, functional analysis of avoidance | `pain-medical.md` (read `safety.md`) |
| Behavioral avoidance (the engine) | TRAP/TRAC, Anti-Avoidance Assignments, Graded Task Assignment | `core-techniques.md` |
| Rumination as avoidance | Functional analysis of the rumination + attending-to-experience; treat rumination as the AP in a TRAP | `core-techniques.md` |
| Routine disruption | Routine Establishment (wake-time anchor first), then layer scheduling | `core-techniques.md` + `depression.md` |
| Special population / delivery | Standard BA, adapted per the file (session length, parent/caregiver involvement, format) | `special-populations.md` + `core-techniques.md` |
| Maintenance / nearing the end of the work | Relapse Prevention Plan + ACTION acronym for self-activation | `core-techniques.md` |

## The universal spine

Whatever the area, most first sessions move through the same early arc — don't skip straight to scheduling: **Activity Monitoring** (see what's actually happening and how mood tracks it) comes before **Activity Scheduling** (the workhorse), and avoidance gets named with **TRAP/TRAC** when it surfaces. The monitoring data, not your clinical guess, picks the scheduling targets. If even the smallest scheduled step won't go, that's **Graded Task Assignment**, not failure.

## Multi-problem and re-routing

- Most sessions are one area; some are layered (low mood *and* anhedonia; depression *and* a specific avoidance; grief *and* rumination). Work one at a time, the most acute first. Name the others and offer to come back.
- If a walk-through reveals the real problem is different from where you started, say so plainly and switch files. Don't finish the wrong technique for tidiness.
- **"Pointless" splits two ways:** pointless about *activities/effort* is anhedonia/ordinary depression, in scope → `depression.md` / `anhedonia.md`. "No point going on / in being here" is about their life — that's a crisis trigger, drop the technique and follow `safety.md`. When genuinely ambiguous, ask the one calm screening question in `safety.md` first.
- **Avoidance vs a real external barrier:** if the block is genuine circumstance (no childcare, no money for the thing), that's **Problem-Solving** (`core-techniques.md`); if the block is the depression-driven pull to withdraw, that's **TRAP/TRAC**. Don't problem-solve avoidance or TRAC a real logistics problem.
- When the routed area is clinician-guided (severe depression with no initiative, trauma-related avoidance, substance use, postpartum), deliver only the self-help-safe layer and say so (`safety.md`).
